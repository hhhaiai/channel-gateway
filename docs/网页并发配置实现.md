# 网页并发配置实现

> 状态：实施文档
>
> 范围：在现有 Channel Gateway 控制台读取和保存全局 Delivery Worker 并发配置。

## 1. 控制台展示

连接成功后，控制台通过认证接口读取：

```json
{
  "deliveryMaxConcurrency": null,
  "effectiveDeliveryMaxConcurrency": 16,
  "deliveryMaxConcurrencySource": "detected",
  "deliveryMaxConcurrencyHardMax": 256,
  "resources": {
    "cpuCount": 8,
    "memoryLimitBytes": 8589934592,
    "memorySource": "constraint"
  }
}
```

字段含义：

- `deliveryMaxConcurrency`：持久化显式覆盖；`null` 表示自动。
- `effectiveDeliveryMaxConcurrency`：下次启动实际使用的解析结果。
- `source`：`config`、`environment` 或 `detected`。
- `resources`：启动时进程可见的 CPU 和内存快照。

## 2. 保存显式值

```json
{
  "links": [],
  "revision": "<sha256>",
  "deliveryMaxConcurrency": 12
}
```

只接受 `1..256` 的整数。保存后写入 OpenClaw source config：

```text
plugins.entries.channel-gateway.config.deliveryMaxConcurrency
```

## 3. 恢复自动模式

```json
{
  "links": [],
  "revision": "<sha256>",
  "deliveryMaxConcurrency": null
}
```

服务删除持久化 override。下次启动重新按照：

```text
环境变量 > CPU/内存探测
```

解析有效值。

## 4. 原子配置和冲突保护

links 与并发 override 属于同一个编辑文档，共享 revision。两个管理员同时修改时，后保存者收到 `CONFIG_CONFLICT`，必须重新加载，不能覆盖先前修改。

旧客户端如果 PUT 时不包含 `deliveryMaxConcurrency`，服务保留当前值，不会意外清除。

## 5. 重启语义

保存只修改 source config，不热替换当前 worker。响应始终包含：

```json
{ "restartRequired": true }
```

这样避免运行中的 lane 数量突然改变、shutdown drain 语义不明确或部分请求使用不同上限。

## 6. 安全边界

- 页面继续使用 Gateway Bearer token。
- Token 只保存在当前页面内存。
- 页面输入校验不能替代服务端校验。
- 服务端复用统一 `1..256` 解析逻辑。
- 资源信息只包含 CPU 数量和内存上限，不包含环境变量、文件路径或凭据。

## 7. 不包含范围

- 单平台/单账号速率限制。
- 429 cooldown 和熔断。
- 动态热调整 worker。
- 高频消息聚合和 AI 摘要。
- 异常负责人通知。
